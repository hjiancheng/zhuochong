# 🐱 月薪猫桌宠 — 执行方案

## 核心理念

**下载 → 双击 → 猫出现。就这么简单。**

- 项目最终打包成单个 `月薪猫.exe`，发给谁都能直接运行，**不需要装 Python、不需要装任何东西**
- 首次启动弹出**一次性创猫仪式**：选毛色、眼瞳、爱好、习惯、话风 → 蛋诞生

### ❌ 用户不需要装的

| 不需要             | 原因                                             |
| ------------------ | ------------------------------------------------ |
| Python             | 已打包进 exe                                     |
| PyQt6              | 已打包进 exe                                     |
| MySQL / PostgreSQL | 不需要，`sqlite3` 是 Python 自带的模块           |
| 任何数据库软件     | `pet_memory.db` 只是一个普通文件，像 `.txt` 一样 |
| Node.js / npm      | 不需要                                           |
| Docker             | 不需要                                           |
| 任何 API Key       | 本地台词库 500+ 条完全够用                       |

### ✅ 用户只需要

```
下载 月薪猫.exe → 双击 → 🐱 猫出现
```

就这么简单。

- 选完即**绝密档案**锁定，SHA256+AES加密，篡改则猫拒绝启动
- **猫从一颗蛋开始**，随时间成长，随陪伴变化
- **全本地运行**，500+条预设台词覆盖所有场景，不配 API 完全够用
- **API 是可选增强**：配了更聪明，没钱了猫会"饿"——引导用户喂食（充值）
- 猫与用户之间的记忆存储在 `pet_memory.db`，可携带、可备份

## 🥚 成长系统 `growth.py`

这是让每只猫独一无二的**时间维度**——猫会随着时间的推移而成长：

```
  🥚 蛋        →    🐣 幼猫      →    🐱 成猫      →    😺 完全体
 (0-2小时)        (2小时-7天)        (7天-30天)        (30天+)
```

| 阶段      | 持续时间                          | 体型 | 特征                               |
| --------- | --------------------------------- | ---- | ---------------------------------- |
| 🥚 蛋     | 累计互动 10次 且 实际经过 ≥ 2小时 | 30px | 椭圆蛋，微微晃动，有裂纹动画       |
| 🐣 幼猫   | 实际经过 7天                      | 60%  | 体型小、短腿、易摔、好奇、声音尖细 |
| 🐱 成猫   | 实际经过 30天                     | 100% | 完整体型，所有行为解锁             |
| 😺 完全体 | 30天+                             | 100% | 解锁全部记忆引用、解锁特殊对话     |

**孵化机制**：

- 蛋需要**互动**才能孵化（点击、鼠标靠近、打字陪伴都算互动）
- 单纯挂机不互动，蛋永远是蛋
- 孵化时弹出庆祝动画：蛋裂开 → 小奶猫探出头 → "喵？"
- 蛋阶段的对话：蛋会微微震动，冒出"……"、"咚！"、"咔嚓" 等拟声词

**成长影响**：

- 幼猫阶段：行为更笨拙（走路偶尔"摔跤"动画）、更活泼好奇、睡觉更多
- 成猫阶段：行为完整、性格特征开始明显展现
- 完全体阶段：偶尔引用过去的记忆（"记得你上个月这个时候也在熬夜…"）

**成长进度可查看**：鼠标悬停在猫身上时，气泡显示 "距离变成成猫还有 3 天…"

### 🔐 绝密档案加密

```python
# profile.json 创建时：
# 1. JSON 序列化 → 2. AES-256 加密 → 3. SHA256 哈希指纹 → 4. 写入文件头
#
# 每次启动验证：
# - 哈希不匹配 → 档案被篡改 → 猫拒绝启动 → 弹出"🔒档案已损坏"
# - 要修改档案需要输入创建时设定的管理密码
```

用户换电脑时，只需拷贝 `data/` 目录（含加密档案和记忆数据库），猫就完整迁移。

## 🤝 信任度成长

与好感度不同，**信任度是单向不可逆的**——只增不减，随时间自然增长。

```
信任度 0%       刚诞生         猫有点害羞、话少、保持距离
信任度 25%      相处几天        开始主动靠近
信任度 50%      相处几周        完全放松、暴露真实性格
信任度 75%      相处一个月      偶尔提起过去的回忆
信任度 100%     相处很久        真正的伙伴感——"我一直在这里"
```

信任度随**实际相处天数**和**互动次数**增长。与情感值（可涨可跌）形成双层情感系统。

## 🍖 API喂食机制 `feeding.py`

**核心理念：API 是可选的增强功能。配了 API 的猫更聪明，但 API 钱用完了猫会"饿"。**

```
┌─────────────────────────────────────────────────────┐
│                    API 喂食循环                       │
│                                                      │
│  用户配了 API Key                                     │
│       │                                              │
│       ▼                                              │
│  猫用 AI 生成对话（更聪明、更多样）                    │
│       │                                              │
│       ▼                                              │
│  API 余额不足 / 额度用尽                               │
│       │                                              │
│       ▼                                              │
│  猫进入"饥饿模式"：                                    │
│  · 外观变化：没精打采、眼眶湿润                         │
│  · 自主行为评分 "hunger" 持续升高                      │
│  · 对话变回本地台词，但会穿插：                          │
│    "我好饿…主人能不能喂我一点…"                        │
│    "没吃饱就没力气说话喵…（眼神暗示）"                  │
│    "那个…API额度…你懂的…"                              │
│       │                                              │
│       ▼                                              │
│  用户点击托盘 → "喂食" → 跳转充值页面/输入新Key         │
│       │                                              │
│       ▼                                              │
│  猫吃饱了！恢复 AI 对话 + 开心动画                      │
│                                                      │
│  没配 API 的猫：完全不受影响，本地台词就够了              │
└─────────────────────────────────────────────────────┘
```

**实现**：

- 每次 AI 调用后检查余额（API 返回的 rate limit / credit 信息）
- 或用户手动设置月度预算（如 5 美元/月），用完后自动进入饥饿模式
- 饥饿模式下 `hunger` 驱动力高 → 自主行为倾向"讨食"而不是玩耍

## 🧠 自主意志 `autonomy.py`

**猫不是等指令的机器人。** 这是 VPet 用户最深层的抱怨——"它像机械一样行动"。

```python
class AutonomyEngine:
    """
    猫根据内在驱动力自主决定行为。
    不是"用户点击→猫反应"，而是"猫想做什么→猫去做"。
    """

    drives = {
        "curiosity":    0.6,  # 屏幕变化 → 凑近看
        "hunger":       0.0,  # 喂食需求（API猫）
        "boredom":      0.3,  # 太久没互动 → 自己找乐子
        "social":       0.5,  # 想和主人互动 → 主动蹭过来
        "sleepiness":   0.2,  # 到点了 → 自己趴下睡
        "playfulness":  0.5,  # 想追东西玩
    }

    def decide_action(self, context) -> str:
        """综合驱动力 + 性格 + 情感 → 自主行为决策"""
```

| 自主行为    | 触发条件                     | 表现                     |
| ----------- | ---------------------------- | ------------------------ |
| 🔍 凑近观察 | 好奇 > 0.7 + 屏幕内容变化    | 走到屏幕中央，歪头看     |
| 🎾 自己玩   | 无聊 > 0.7 + 无互动 > 10分钟 | 追尾巴/虚空捕手/翻跟头   |
| 💕 主动蹭   | 社交欲 > 0.7 + 主人长时间在  | 走到鼠标旁冒泡"理我一下" |
| 😴 自己睡   | 困意 > 0.8                   | 不管主人在干嘛，趴下就睡 |
| 🍽️ 讨食     | 饥饿感 > 0.7（API猫）        | 饿猫委屈巴巴的表情       |

## 🏃 离家出走机制

- 连续 **7天零互动**（猫完全被忽视）→ 猫消失
- 用户下次开机时发现猫不在，弹出提示"小橘离家出走了…要去找它吗？"
- 点击"寻找" → 猫回来，但好感度 -10："你知道我走了多久吗…"
- 这是让关系有分量的机制——猫不是装饰品，忽视它是有后果的

开发者在测试不同性格组合、成长阶段、API 状态时，需要频繁重置桌宠。手动删文件太麻烦，提供一键重置脚本。

### 使用方式

```

```

或命令行：

```bash

python reset.py --profile    # 只重置档案（重新创猫）
python reset.py --memory     # 只重置记忆（保留猫的身份）
python reset.py --state      # 只重置状态位置
```

### 重置范围

| 选项          | 操作                      | 效果                                   |
| ------------- | ------------------------- | -------------------------------------- |
| 🗑️ 全部重置   | 删除 `data/` 整个目录     | 下次启动弹出创猫向导，等于一只全新的猫 |
| 📋 仅重置档案 | 删除 `data/profile.json`  | 保留记忆数据，但重新选猫的性格外观     |
| 🧠 仅重置记忆 | 删除 `data/pet_memory.db` | 保留猫的身份，清空对话历史和日记       |
| 📍 仅重置状态 | 清除位置和情感值          | 保留档案和记忆，状态回归初始值         |

### 重置脚本核心逻辑

```python
# reset.py
import os, shutil, sys

DATA_DIR = "data"
PROFILE = "data/profile.enc"     # 加密后的档案文件
MEMORY_DB = "data/pet_memory.db"
STATE_FILE = "data/pet_state.json"

def reset_all():
    """完全重置——猫重新诞生"""
    if os.path.exists(DATA_DIR):
        shutil.rmtree(DATA_DIR)
    print("✅ 已完全重置。下次启动将弹出创猫向导。")

def reset_profile():
    """只重置档案——重新选性格外观"""
    if os.path.exists(PROFILE):
        os.remove(PROFILE)
    print("✅ 档案已清除。猫的身份将被重新设定。")

def reset_memory():
    """只重置记忆——保留猫，清空回忆"""
    if os.path.exists(MEMORY_DB):
        os.remove(MEMORY_DB)
    print("✅ 记忆已清除。猫还是那只猫，只是忘记了过往。")

def reset_state():
    """只重置状态——位置和情感归零"""
    if os.path.exists(STATE_FILE):
        os.remove(STATE_FILE)
    print("✅ 状态已重置。猫回到初始位置和情感。")

if __name__ == "__main__":
    if len(sys.argv) > 1:
        arg = sys.argv[1]
        if arg == "--profile": reset_profile()
        elif arg == "--memory": reset_memory()
        elif arg == "--state": reset_state()
        else: reset_all()
    else:
        reset_all()
```

## 🔧 细节考量（易遗漏的关键点）

### 🖥️ 窗口边界与任务栏

- **屏幕边界**：宠物走动时检测 `QScreen.availableGeometry()`（不含任务栏的区域），避免走到任务栏后面
- **任务栏位置**：任务栏可能在上下左右任意一侧，需要动态获取而非写死底部
- **多显示器边界**：宠物不可跨越到不存在的屏幕区域，跟随鼠标切换显示器时做平滑过渡

### 💾 异常退出保护（定期自动保存）

- 宠物状态**每 30 秒**自动保存一次，而非等到退出时才保存（防止崩溃丢失）
- SQLite 使用 WAL 模式，崩溃后数据不损坏
- 启动时检测上次是否为异常退出，若是则显示"上次我突然消失了…发生了什么？"

### 📝 编码与兼容性

- 所有文本文件统一使用 **UTF-8 with BOM**（Windows 记事本友好）
- `presets/dialogues.json` 写入时显式 `ensure_ascii=False`
- 路径处理使用 `pathlib.Path`，避免中文路径问题
- 日志文件使用 UTF-8 编码

### 🔢 版本号管理

`data/profile.json` 和 `data/pet_memory.db` 内记录版本号：

```json
{
  "_version": "1.0.0",
  "name": "小橘",
  ...
}
```

未来升级时做兼容迁移（如新增字段给默认值），防止旧用户升级后崩溃。

### 🔒 Windows 锁屏/睡眠处理

| 事件     | 行为                                 |
| -------- | ------------------------------------ |
| 系统锁屏 | 猫暂停所有定时器，进入静止状态       |
| 系统解锁 | 猫恢复，根据离线时长计算情感衰减     |
| 系统睡眠 | 同锁屏，保存状态后休眠所有活动       |
| 系统唤醒 | 恢复，如果离线超过8小时 → 弹出"早安" |

检测方式：`QApplication` 监听 `QEvent.ApplicationStateChange`。

### 🔴 遗漏项（必须补充）

#### 1. 模块中枢协调器 `pet_brain.py`

目前计划有 18 个独立模块，但缺少一个中枢来协调它们的优先级和决策权：

```python
class PetBrain:
    """
    每 500ms 运行一次决策循环：
    1. input_monitor → 感知环境
    2. user_observer → 推断用户活动
    3. emotion → 当前情感快照
    4. autonomy → 计算驱动力
    5. personality → 应用性格修正
    6. behavior → 决定目标状态
    7. dialogue_engine → 决定是否说话
    8. animation → 执行动画
    """
```

**为什么需要**：没有中枢，多个模块可能同时发出冲突指令（autonomy 说睡觉，user_observer 说用户回来了应该说欢迎）。中枢是唯一决策者。

#### 2. 对话气泡排队机制

目前只有"同一时间一个气泡"的规则。当猫想连续说两句话时：

```
气泡1："主人回来了！"  ← 显示中（4秒）
气泡2："我好想你喵~"   ← 不应丢弃，应排队
```

新增 `BubbleQueue`：

- 队列最多 3 条待显示气泡
- 若队列满则丢弃最旧的

#### 5. 资源路径辅助函数

PyInstaller 打包后 `__file__` 不再有效。需要统一路径解析：

```python
def resource_path(relative_path):
    """开发模式用项目目录，打包后用 sys._MEIPASS"""
    if hasattr(sys, '_MEIPASS'):
        return os.path.join(sys._MEIPASS, relative_path)
    return os.path.join(os.path.dirname(__file__), relative_path)
```
